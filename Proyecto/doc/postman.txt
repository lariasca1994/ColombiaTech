Puedes utilizar estos JSON en Postman para probar las operaciones CRUD del modelo House. Asegúrate de configurar la URL correcta de acuerdo a tu servidor local o remoto, y el método HTTP correspondiente (POST, GET, PUT, DELETE) para cada operación.

Por ejemplo, si tu servidor está ejecutándose en http://localhost:3000/graphql, puedes utilizar las siguientes URLs en Postman:

Crear una nueva casa (POST): http://localhost:3000/graphql
Obtener todas las casas (GET): http://localhost:3000/graphql?query={houses{id,address,city,state,size,type,zip_code,code,rooms,bathrooms,price,image}}
Obtener una casa por ID (GET): http://localhost:3000/graphql?query={house(id:"ID_DE_LA_CASA"){id,address,city,state,size,type,zip_code,code,rooms,bathrooms,price,image}}
Actualizar una casa (PUT): http://localhost:3000/graphql
Eliminar una casa (DELETE): http://localhost:3000/graphql
Recuerda que para las operaciones de mutación (crear, actualizar y eliminar), debes incluir la mutación correspondiente en el cuerpo de la solicitud GraphQL.

Estas pruebas te permitirán verificar que las operaciones CRUD del modelo House estén funcionando correctamente en tu aplicación NestJS.
